# =============================================================================

#
# NumPy 1.26.4: última versão da série 1.x, compatível com TensorFlow 2.16.1.
# NumPy 2.0+ QUEBRA compatibilidade com TensorFlow < 2.17. NÃO usar.

# =============================================================================
numpy==1.26.4               # Arrays, FFT, operações vetorizadas. DEP: NÃO usar 2.0+
scipy==1.11.4               # Filtros IIR, resample_poly, detrend, find_peaks

# =============================================================================
# 4. MANIPULAÇÃO DE DADOS (Pandas + PyArrow)

#
# Pandas 2.1.4: última estável da série 2.1, compatível com NumPy 1.26.
# PyArrow 14.0.1: última estável antes da série 15 (que mudou API de tipos).
# =============================================================================
pandas==2.1.4               # DataFrames para features e anotações AAMI
pyarrow==14.0.1             # Backend Parquet para cache de datasets (opcional)

# =============================================================================
# 5. VISUALIZAÇÃO E EDA (Matplotlib + Seaborn)
# =============================================================================
# Usados APENAS em notebooks EDA e debug. Não entram no pipeline de produção.

# Seaborn 0.13.0: estatísticas visuais (distribuições de RR-interval, etc.).
# =============================================================================

# =============================================================================
# 6. MACHINE LEARNING (TensorFlow + Keras)
# =============================================================================
# TensorFlow 2.16.1: última estável com Keras 3 integrado e TFLite robusto.
# Fornece: 1D-CNN, treinamento, conversão TFLite, quantização PTQ (INT8).
#
# RESTRIÇÕES CRÍTICAS:
# - NÃO usar TensorFlow 2.17+ sem validar TFLM compatibilidade (tflite-micro).
# - NÃO usar TensorFlow 2.15.x (Keras 2, API diferente de Keras 3).

# - TFLite converter em TF 2.16 suporta INT8 PTQ com representative_dataset.
#
# Dependências transitivas importantes:
#   - h5py: salvar/ler modelos .h5
#   - protobuf: serialização de grafos TF

# =============================================================================

# =============================================================================
